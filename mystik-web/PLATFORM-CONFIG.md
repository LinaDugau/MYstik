## 🔍 Диагностика проблемы

### Проблема: `lstat /workspace/app/mystik-web: no such file or directory`

**Причина:** Платформа ищет неправильный путь к проекту

**Решение:**
1. Убедитесь, что в настройках платформы указан путь: `mystik-web`
2. Dockerfile должен находиться в `mystik-web/Dockerfile`
3. Структура репозитория должна быть:
   ```
   repository-root/
   ├── mystik-web/
   │   ├── Dockerfile
   │   ├── package.json
   │   ├── src/
   │   ├── server/
   │   └── ...
   ├── app/ (мобильное приложение)
   └── другие файлы
   ```

### Проверка структуры

Выполните в корне репозитория:
```bash
ls -la mystik-web/
```

Должно показать:
```
Dockerfile
package.json
src/
server/
dist/ (после сборки)
```

## 🆘 Устранение проблем

### Если контейнер не запускается:
1. Проверьте логи: `docker logs mystik-web-app`
2. Проверьте Dockerfile синтаксис
3. Убедитесь, что все файлы скопированы правильно

### Если API возвращает HTML:
1. Проверьте, что используется `production-server.js`
2. Убедитесь, что миграция выполнилась
3. Проверьте порядок middleware в сервере

### Если нет доступа к приложению:
1. Проверьте настройки сети
2. Убедитесь, что порт 8000 открыт
3. Проверьте IP адрес контейнера
