# 🔍 Диагностика проблем API

## Быстрая диагностика

### 1. Тестовая страница API
Откройте в браузере: `https://linadugau-mystik-39d3.twc1.net/test-api.html`

Эта страница протестирует все основные API endpoints и покажет детальную информацию о каждом запросе.

### 2. Проверка логов сервера
После деплоя проверьте логи сервера. Теперь они содержат:
- Все входящие запросы с заголовками
- CORS проверки с детальной информацией
- Результаты запросов к базе данных
- Ошибки API с полной информацией

### 3. Проверка консоли браузера
В консоли браузера теперь отображается:
- Используемый API_BASE_URL
- Статус каждого API запроса
- Content-Type ответов
- Первые 200 символов ответа при ошибках

## Возможные проблемы и решения

### Проблема: "Failed to fetch"
**Причины:**
1. CORS блокирует запросы
2. Сервер недоступен
3. Неправильный API_BASE_URL

**Диагностика:**
1. Проверьте консоль браузера на CORS ошибки
2. Откройте `test-api.html` для детальной диагностики
3. Проверьте логи сервера на CORS проверки

### Проблема: "Unexpected token '<', \"<!DOCTYPE \"... is not valid JSON"
**Причины:**
1. API endpoint возвращает HTML вместо JSON
2. Статические файлы обрабатываются до API роутов
3. API endpoint не существует

**Диагностика:**
1. Проверьте Content-Type в консоли браузера
2. Проверьте логи сервера на порядок обработки запросов
3. Убедитесь, что API endpoints определены до статических файлов

### Проблема: "Error loading quizzes/tarot"
**Причины:**
1. База данных не инициализирована
2. Миграция не выполнилась
3. Ошибка в SQL запросах

**Диагностика:**
1. Проверьте логи сервера на ошибки базы данных
2. Запустите `node test-endpoints.js` для проверки БД
3. Проверьте результат миграции в логах

## Команды для диагностики

### Локальная диагностика:
```bash
# Проверка базы данных
cd mystik-web/server
node test-endpoints.js

# Проверка содержимого базы данных
node check-db.js

# Запуск миграции
npm run migrate
```

### Проверка API endpoints:
```bash

curl https://linadugau-mystik-39d3.twc1.net/api/health
```
