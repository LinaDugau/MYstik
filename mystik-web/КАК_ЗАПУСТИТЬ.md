# 🚀 Как запустить Mystik Web

## 📋 Требования

- **Node.js** версии 18 или выше ([скачать](https://nodejs.org/))
- **Bun** для запуска сервера ([инструкция ниже](#установка-bun))

## 🎯 Самый простой способ (рекомендуется)

### Windows (PowerShell):
```powershell
cd mystik-web
.\start.ps1
```

### macOS/Linux (Bash):
```bash
cd mystik-web
./start.sh
```

Выберите вариант **3** - запустит и сервер, и клиент одной командой!

## Варианты запуска

### Вариант 1: Всё одной командой

**Windows:**
```powershell
cd mystik-web
.\start.ps1
```

**macOS/Linux:**
```bash
cd mystik-web
./start.sh
```

Выберите:
- **1** - Только сервер (порт 3001)
- **2** - Только клиент (порт 5173)
- **3** - Оба сразу (сервер в фоне, клиент в текущем терминале)

### Вариант 2: Вручную

**Терминал 1 - Сервер:**

Windows:
```powershell
cd mystik-web\server
$env:USERPROFILE\.bun\bin\bun.exe --watch index.js
```

macOS/Linux:
```bash
cd mystik-web/server
~/.bun/bin/bun --watch index.js
```

**Терминал 2 - Клиент:**
```bash
cd mystik-web
npm run dev
```

## Адреса

- 🖥️ Сервер: http://localhost:3001
- 🌐 Клиент: http://localhost:5173

## Установка Bun

### Windows:
```powershell
powershell -c "irm bun.sh/install.ps1 | iex"
```

### macOS/Linux:
```bash
curl -fsSL https://bun.sh/install | bash
```

После установки перезапустите терминал.

## Если bun не найден

### Windows:
Проверьте, что bun установлен:
```powershell
$env:USERPROFILE\.bun\bin\bun.exe --version
```

### macOS/Linux:
Добавьте в текущую сессию терминала:
```bash
export PATH="$HOME/.bun/bin:$PATH"
```

Или перезапустите терминал (bun уже добавлен в ~/.zshrc или ~/.bashrc)

## Проверка работы

1. Откройте http://localhost:5173
2. Перейдите в профиль
3. Нажмите "Редактировать"
4. Измените имя или дату рождения
5. Нажмите "Сохранить"

Теперь всё должно работать без ошибок CORS! ✨

## Остановка

Если запустили через start.sh (вариант 3):
- Нажмите `Ctrl+C` чтобы остановить клиент
- Сервер остановится автоматически или найдите его PID в выводе и выполните: `kill <PID>`
